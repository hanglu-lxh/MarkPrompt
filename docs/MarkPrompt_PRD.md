# MarkPrompt 产品需求文档

> 产品版本：V1 单版本规划  
> 原型基准：V4 高保真交互稿  
> 目标平台：macOS 原生应用  
> 文档目标：定义 MarkPrompt 的相对终态产品形态，作为设计和开发实现依据。

![MarkPrompt V4 原型](assets/markprompt_interaction_prototype_v4.png)

---

## 1. 产品定位

MarkPrompt 是一个本地优先的 macOS Markdown 审稿工具，用于把用户在文档阅读过程中的画线、批注和修改判断，转化为可以直接交给 Codex、Claude Code 等 AI 编程工具执行的精准 Prompt。

它不是完整 Markdown 编辑器，也不是通用 AI 聊天工具。它解决的是一个更具体的问题：

> 用户已经知道文档哪里需要改，但需要一个足够顺手、足够精确的方式，把这些局部修改意图交给 AI 执行。

核心价值是把“人类审稿判断”变成“AI 可执行修改指令”。

---

## 2. 核心场景

### 2.1 主要场景

用户让 Codex 或其他 AI 生成一份 Markdown 文档后，需要进行二次审稿和修改。相比在对话里笼统描述“帮我改第几段”，用户更希望像读文档一样直接在内容上画线、批注，然后一键生成可复制的修改 Prompt。

典型流程：

```text
打开 Markdown
  ↓
阅读文档
  ↓
选中文本
  ↓
点击浮动“批注 +”
  ↓
输入批注意见或点击快捷提示
  ↓
右侧自动汇总批注
  ↓
实时生成 Prompt 预览
  ↓
复制 Prompt 给 Codex 执行修改
```

### 2.2 目标用户

- 高频使用 Codex、Claude Code、Cursor 等 AI 工具的人；
- 产品经理、创业者、独立开发者、方案作者、内容创作者；
- 经常处理 PRD、方案、研究报告、技术说明、复盘文档的人；
- 对文档质量有明确判断，但不想手动逐段重写的人。

### 2.3 用户目标

- 像读文档一样自然地发现问题；
- 像审稿一样直接标出需要修改的位置；
- 用自然语言表达修改意见；
- 减少写 Prompt 和定位上下文的成本；
- 让 AI 更准确地修改原始 Markdown 文件。

## 3. 产品原则

### 3.1 一窗完成

MarkPrompt 的核心工作都在一个主窗口内完成。批注创建、批注管理、Prompt 预览和复制都不应跳到独立管理弹窗。

允许存在轻量浮层：

- 选区旁的“批注 +”浮动按钮；
- 由浮动按钮触发的添加批注小弹框；
- 系统级保存、错误、确认提示。

不允许存在：

- 单独的批注管理大弹窗；
- 单独的 Prompt 生成大弹窗；
- 需要用户反复切换窗口的主流程。

### 3.2 用户文本优先

批注最终以用户在输入框中写下的自然语言为准。快捷提示按钮只是辅助填充，不替代用户判断。

界面中不暴露“类型、范围、强度”等复杂表单字段。系统可以在后台做结构化推断，但不能让用户为此增加操作负担。

### 3.3 阅读不中断

用户的主要心流是阅读和审稿。添加批注必须轻量，浮层要贴近选区，保存后立即回到阅读状态。

### 3.4 Prompt 可信

生成的 Prompt 必须清楚说明：

- 要修改哪个文件；
- 哪些文本被批注；
- 用户对每处文本的意见是什么；
- AI 应该如何处理冲突、保持结构和输出结果。

## 4. 产品形态

### 4.1 平台形态

MarkPrompt 首发为原生 macOS 应用。

建议技术方向：

- SwiftUI 负责主窗口、工具栏、侧边栏、右侧检查器；
- AppKit / TextKit 负责正文渲染、文本选择、选区定位和高亮；
- swift-markdown 负责 Markdown 结构解析；
- 本地 JSON sidecar 文件保存批注；
- NSPasteboard 负责复制 Prompt；
- UserDefaults 或本地配置文件保存用户偏好。

### 4.2 主窗口布局

V4 原型采用一窗三栏布局：

```text
┌──────────────────────────────────────────────────────────────┐
│ macOS Toolbar                                                 │
├──────────────┬───────────────────────────────┬───────────────┤
│ 大纲 Sidebar  │ Markdown 阅读区                │ 批注与 Prompt   │
│ Outline      │ Reader                         │ Inspector      │
└──────────────┴───────────────────────────────┴───────────────┘
```

主窗口包含：

- 顶部工具栏；
- 左侧文档大纲；
- 中间 Markdown 阅读区；
- 右侧“批注与 Prompt”检查器；
- 底部轻量状态栏。

### 4.3 顶部工具栏

工具栏需要保持原生、克制、轻量。

包含：

- macOS traffic light；
- 侧栏切换按钮；
- 打开文件按钮；
- 居中文档标题，例如 `MarkPrompt - 产品需求文档.md`；
- 搜索框；
- 设置入口；
- 主操作按钮 `复制 Prompt`。

### 4.4 左侧大纲

左侧大纲用于帮助用户在长文档中定位结构。

能力：

- 自动提取 H1-H6 标题；
- 当前阅读位置高亮；
- 点击大纲项跳转到对应章节；
- 支持折叠章节；
- 底部保留大纲、搜索、筛选等轻量入口。

### 4.5 中间阅读区

中间区域是核心阅读和批注空间。

要求：

- 渲染 Markdown，而不是显示原始 Markdown 文本；
- 标题、段落、列表、代码块、引用、表格具备清晰样式；
- 支持文本选择；
- 支持已批注文本的高亮和下划线；
- 选区旁显示轻量浮动批注入口；
- 不因右侧面板刷新而影响阅读滚动位置。

### 4.6 右侧批注与 Prompt 面板

右侧面板是唯一的批注管理和 Prompt 生成区域。

顶部：

- 标题 `批注与 Prompt`；
- 分段控件 `批注 / Prompt`；
- 筛选或设置图标。

主体：

- 批注卡片列表；
- Prompt 预览区；
- 底部操作按钮。

关键要求：

- 批注和 Prompt 都在同一右侧面板内解决；
- 默认状态下显示批注列表，同时保留 Prompt 预览；
- 用户不需要打开独立弹窗即可复制或保存 Prompt；
- 批注变化后 Prompt 预览实时更新。

## 5. 核心交互

### 5.1 添加批注

用户在阅读区选中文本后，选区右侧或末端出现小型浮动按钮：

```text
[批注 +]
```

点击后出现添加批注小弹框。弹框锚定在选区附近，不离开阅读上下文。

弹框内容：

- 标题：`添加批注`；
- 输入框标签：`批注意见`；
- 多行文本输入框；
- 快捷提示按钮；
- 底部 `取消` 和 `保存`。

快捷提示按钮：

- `优化表达`
- `重写这段`
- `优化语气`
- `补充措施`
- `强化论证`
- `压缩精简`

不展示：

- 类型；
- 范围；
- 强度；
- 复杂标签表单。

### 5.2 快捷提示

快捷提示按钮用于向批注意见输入框插入可编辑文本。

默认填充规则：

| 按钮 | 插入文本 |
|---|---|
| 优化表达 | 请优化这段表达，让它更清晰、更直接，但保持原意。 |
| 重写这段 | 请重写这段内容，保留核心观点，但让逻辑和表达更顺。 |
| 优化语气 | 请调整这段语气，让它更专业、更克制。 |
| 补充措施 | 请补充更具体的行动措施或落地方式。 |
| 强化论证 | 请加强这段论证，让观点和结论更有说服力。 |
| 压缩精简 | 请压缩这段内容，去掉重复和不必要的表达。 |

交互规则：

- 输入框为空时，点击按钮直接填入文本；
- 输入框已有内容时，点击按钮追加到新行；
- 用户可以继续编辑插入文本；
- 保存时以输入框最终内容为准；
- 多个快捷提示可以组合使用。

### 5.3 批注卡片

批注保存后，右侧面板立即出现批注卡片。

卡片展示：

- 批注编号，例如 `note_001`；
- 选中文本摘要；
- 批注意见；
- 快捷提示标签；
- 是否纳入 Prompt 的开关；
- 更多操作入口。

用户可以：

- 点击卡片跳转到正文位置；
- 编辑批注意见；
- 删除批注；
- 切换是否纳入 Prompt；
- 查看定位状态。

### 5.4 Prompt 预览

Prompt 预览位于右侧同一面板下方，不使用单独弹窗。

预览内容根据当前有效批注实时生成。用户可以：

- 快速扫视 Prompt 是否符合预期；
- 点击 `复制 Prompt`；
- 点击 `保存 .prompt.md`；
- 切换 Prompt 模板；
- 在 Prompt 分段视图中查看更完整内容。

### 5.5 保存与状态

应用需要自动保存审稿会话。

默认文件：

```text
example.md
example.review.json
example.prompt.md
```

底部状态栏展示：

- 字数；
- 当前语言；
- 缩放比例；
- 行列位置；
- 保存状态。

## 6. 功能需求

### FR-001 打开 Markdown

用户可以通过菜单、Finder 打开方式、拖拽、最近文件或剪贴板打开 Markdown。

验收标准：

- 支持 `.md` 和 `.markdown`；
- 正确显示文件名；
- 可以读取 UTF-8 文档；
- 打开失败时展示明确错误；
- 打开后自动加载同名 `.review.json`。

### FR-002 渲染阅读视图

应用需要将 Markdown 渲染为适合审稿的阅读视图。

验收标准：

- 支持标题、段落、列表、引用、代码块、行内代码、链接、表格；
- 标题层级清晰；
- 长文滚动顺畅；
- 文本选择不破坏排版；
- 浅色和深色模式均可读。

### FR-003 文档大纲

应用自动解析标题并生成左侧大纲。

验收标准：

- 支持 H1-H6；
- 点击大纲可以跳转；
- 当前阅读章节高亮；
- 无标题文档也能正常阅读。

### FR-004 文本选择

用户可以在渲染后的阅读区选择文本。

验收标准：

- 支持跨行选择；
- 支持段落内选择；
- 选区稳定，不因浮动按钮出现而丢失；
- 选区可用于生成批注锚点。

### FR-005 选区浮动按钮

用户选中文本后出现浮动批注按钮。

验收标准：

- 浮动按钮贴近选区末端；
- 按钮不遮挡选中文本；
- 滚动或取消选择后按钮自动消失；
- 点击后打开添加批注弹框。

### FR-006 添加批注弹框

添加批注使用小型锚定弹框。

验收标准：

- 弹框内只包含批注意见、快捷提示和操作按钮；
- 不展示类型、范围、强度；
- 输入框支持多行；
- 空意见保存时给出提醒；
- 保存后弹框关闭，阅读区保持当前位置。

### FR-007 快捷提示按钮

批注弹框内提供少量快捷提示按钮。

验收标准：

- 点击后向输入框插入可编辑文本；
- 不覆盖用户已写内容；
- 最终 Prompt 以输入框内容为准；
- 快捷提示按钮最多两行展示；
- 支持未来自定义。

### FR-008 批注锚点

每条批注需要保存可回溯定位信息。

锚点字段：

- 文件路径；
- 标题路径；
- 选中文本；
- 起止偏移；
- 前后上下文；
- 文档 hash。

验收标准：

- 文档未变时能精确定位；
- 文档轻微变化时能通过上下文恢复；
- 无法恢复时显示 `定位需确认`。

### FR-009 正文标记

被批注文本在正文中有视觉标记。

验收标准：

- 使用轻量高亮或下划线；
- 当前批注更醒目；
- 标记不影响再次选择文本；
- 深浅色模式均清晰。

### FR-010 批注面板

右侧面板展示批注列表。

验收标准：

- 按文档顺序展示；
- 展示选中文本摘要和批注意见；
- 展示快捷提示标签；
- 展示是否纳入 Prompt；
- 点击批注跳转正文。

### FR-011 批注编辑与删除

用户可以修改或删除已有批注。

验收标准：

- 修改后自动保存；
- 修改后 Prompt 预览同步更新；
- 删除后正文标记消失；
- 删除支持撤销或轻量确认。

### FR-012 排除批注

用户可以将单条批注排除在 Prompt 外。

验收标准：

- 开关位于批注卡片上；
- 被排除批注仍保留；
- Prompt 生成时不包含被排除批注；
- 排除状态有明确视觉反馈。

### FR-013 Prompt 预览

右侧面板内提供 Prompt 预览。

验收标准：

- 批注变化后实时更新；
- 预览区可以滚动；
- 可以复制 Prompt；
- 可以保存为 `.prompt.md`；
- 不打开独立生成弹窗。

### FR-014 Prompt 模板

应用内置多个 Prompt 模板。

默认模板：

- Codex 文件修改模式；
- Claude Code 文件修改模式；
- 通用完整 Markdown 模式；
- 局部章节修改模式。

验收标准：

- 默认使用 Codex 文件修改模式；
- 用户可以切换模板；
- 模板选择可记忆；
- 模板输出结构稳定。

### FR-015 自动保存

应用自动保存批注和 Prompt。

验收标准：

- 批注保存到 `.review.json`；
- Prompt 可保存到 `.prompt.md`；
- 源目录不可写时保存到应用数据目录；
- 保存失败时给出提示。

## 7. Prompt 生成规则

### 7.1 Codex 文件修改模式

默认 Prompt 应要求 Codex 直接修改目标 Markdown 文件。

必须包含：

- 目标文件路径；
- 修改原则；
- 批注列表；
- 每条批注的定位信息；
- 用户批注意见；
- 输出要求。

示例结构：

```text
# Codex 文件修改模式

目标文件：
/Users/xxx/Documents/MarkPrompt/产品需求文档.md

全局修改原则：
- 保持 Markdown 标题层级
- 不新增未经证实的信息
- 语言更直接，更易读
- 以用户批注意见为准

批注列表：

[NOTE note_001]
章节：1.2 核心价值
选中文本：MarkPrompt 的核心价值是让批注更精准...
批注意见：请优化这段表达，让它更清晰、更直接，但保持原意。
要求：修改对应文本，保持上下文连贯。

输出要求：
- 直接修改目标文件
- 完成后说明修改了哪些位置
- 如果定位不确定，先说明原因，不要擅自改写无关段落
```

### 7.2 用户意见优先级

Prompt 中的优先级：

1. 用户手写批注意见；
2. 用户点击快捷提示后保留在输入框中的文本；
3. 系统根据快捷提示推断的元数据；
4. 默认模板规则。

系统推断不得覆盖用户文本。

### 7.3 冲突处理

当多条批注互相冲突时，Prompt 应要求 AI：

- 优先保留用户明确写出的意见；
- 如果无法同时满足，给出冲突说明；
- 不要修改未被批注影响的段落；
- 保持 Markdown 结构稳定。

## 8. 数据结构

### 8.1 批注 JSON

```json
{
  "version": "1.0",
  "source_file": "/Users/xxx/Documents/MarkPrompt/产品需求文档.md",
  "source_hash": "sha256:example",
  "notes": [
    {
      "id": "note_001",
      "status": "confirmed",
      "include_in_prompt": true,
      "anchor": {
        "heading_path": ["1. 产品概述", "1.2 核心价值"],
        "selected_text": "MarkPrompt 的核心价值是让批注更精准，让 AI 修改更可控。",
        "start_offset": 1280,
        "end_offset": 1322,
        "context_before": "在与 AI 协作撰写和修改文档的过程中，",
        "context_after": "它帮助用户将审稿判断转为 Prompt。"
      },
      "comment": "请优化这段表达，让它更清晰、更直接，但保持原意。",
      "quick_prompts": [
        {
          "id": "improve_expression",
          "label": "优化表达",
          "inserted_text": "请优化这段表达，让它更清晰、更直接，但保持原意。"
        }
      ],
      "inferred_metadata": {
        "intent": "improve_expression",
        "scope": "selected_text",
        "strength": "light"
      },
      "created_at": "2026-06-26T11:30:00+08:00",
      "updated_at": "2026-06-26T11:30:00+08:00"
    }
  ]
}
```

### 8.2 状态定义

| 状态 | 含义 |
|---|---|
| draft | 已创建但意见未确认 |
| confirmed | 默认有效，进入 Prompt |
| excluded | 保留但不进入 Prompt |
| anchor_lost | 定位需确认 |

## 9. 视觉与体验要求

### 9.1 视觉方向

以 V4 原型为基准，整体应接近真实 macOS 生产力工具：

- 温暖白色和中性灰表面；
- 石墨色文字；
- 克制蓝色主按钮；
- 柔和琥珀色下划线；
- 轻量毛玻璃或原生材质感；
- 细分割线；
- 小阴影；
- 8px 左右圆角。

### 9.2 信息密度

界面需要适合长时间文档工作：

- 不做营销式大标题；
- 不用装饰性插画；
- 不使用强烈渐变或大面积彩色背景；
- 阅读区留白舒适；
- 右侧面板信息紧凑但不拥挤；
- 所有按钮文本必须完整可读。

### 9.3 关键微交互

- 选中文本后浮动按钮淡入；
- 点击快捷提示后输入框获得焦点；
- 保存批注后正文标记和右侧卡片同步出现；
- 切换纳入 Prompt 开关后预览即时更新；
- 复制成功后按钮给出轻量反馈；
- 自动保存状态在底部状态栏可见。

---

## 10. 非目标

V1 不做：

- 多人协同审稿；
- 云端同步；
- 完整 Markdown 编辑器；
- 所见即所得全文编辑；
- AI 在应用内直接改写源文件；
- 评论线程和复杂任务流；
- 企业权限管理；
- 截图 OCR 批注。

---

## 11. 成功标准

产品达到可用状态时，应满足：

- 用户能在 3 秒内打开一份 Markdown 并开始阅读；
- 用户选中文本后能在 1 次点击内进入批注输入；
- 添加一条批注不需要理解复杂字段；
- 批注列表和 Prompt 预览始终在同一右侧面板内；
- 用户能一键复制可交给 Codex 的 Prompt；
- Codex 能根据 Prompt 对原始 Markdown 做出可定位修改；
- 整体体验像原生 Mac 工具一样轻、快、稳。
